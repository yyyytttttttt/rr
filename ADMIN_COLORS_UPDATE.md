# Обновление цветов админ-панели

## Что было сделано

Обновлена цветовая схема личного кабинета администратора согласно требованиям:

### Новые цвета текста:
- **Обычный текст**: `#4F5338` → `var(--admin-text-primary)`
- **Неактивный текст**: `#636846` → `var(--admin-text-secondary)`
- **Активный текст**: `#967450` → `var(--admin-text-accent)`

## Структура файлов

1. **CSS переменные**: `src/app/admin/admin-tokens.css`
   - Все цвета, отступы, размеры шрифтов определены как CSS переменные
   - Файл структурирован для удобной поддержки

2. **Layout админки**: `src/app/admin/layout.tsx`
   - Специальный layout для админ-панели
   - Импортирует admin-tokens.css напрямую

3. **Обновленные компоненты**:
   - Все компоненты в `/admin/_components/`
   - Все панели в `/admin/_panels/`
   - Все модальные окна в `/admin/_modals/`

## Как проверить

1. **Перезапустите dev сервер** (если он был запущен):
   ```bash
   npm run dev
   ```

2. Откройте админ-панель: `http://localhost:3000/admin`

3. Проверьте, что:
   - Неактивные пункты меню имеют цвет `#636846`
   - Активные пункты меню имеют цвет `#967450`
   - Обычный текст имеет цвет `#4F5338`

## Использование переменных

В компонентах используйте:

```tsx
// Неактивный текст
<span className="text-[var(--admin-text-secondary)]">Текст</span>

// Активный текст
<span className="text-[var(--admin-text-accent)]">Активный</span>

// Обычный текст
<span className="text-[var(--admin-text-primary)]">Обычный</span>
```

## Если цвета не применяются

1. Очистите кэш Next.js:
   ```bash
   rm -rf .next
   npm run dev
   ```

2. Проверьте, что в браузере нет кэша CSS (Ctrl+Shift+R для жесткой перезагрузки)

3. Убедитесь, что файл `src/app/admin/layout.tsx` существует и правильно импортирует CSS

## Список всех доступных переменных

См. файл `src/app/admin/admin-tokens.css` для полного списка:
- Цвета текста: `--admin-text-*`
- Цвета фона: `--admin-bg-*`
- Цвета границ: `--admin-border*`
- Цвета статусов: `--admin-status-*`
- Размеры: `--space-*`, `--text-*`, `--radius-*`
